# 第26章：设计师常与团队合作

**来源定位**：PDF 513–525页

## 核心思想

团队通过共同目标、信任、沟通和多样技能形成共享设计能力；深度倾听胜过个人英雄主义。

## 反模式

- **让职位替代信息，让最高声量者替代证据**：它会把设计讨论从玩家体验和证据带回功能、偏好或权威。

## 实践示例

1. 选一个正在设计的具体游戏场景。
2. 应用本章的“团队”检查现状。
3. 写出一个可能推翻当前方案的反例。
4. 用最小原型验证：让每个成员复述体验目标和最大风险，比较差异并消除歧义。

## 决策要点

1. 先写目标体验，再选择实现手段。
2. 将直觉表达为可失败假设。
3. 用玩家行为而非满意度口号判断结果。
4. 改动后检查它对相关系统元素与长期体验的连锁影响。
