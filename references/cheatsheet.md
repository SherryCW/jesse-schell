# 决策速查

| 如果你看到… | 优先做… | 因为… |
|---|---|---|
| 功能很多但不好玩 | 写本质体验句，删掉不支撑它的元素 | 体验而非功能是设计对象 |
| 团队意见冲突 | 写唯一最大风险并做最小原型 | 证据比声量更能收敛 |
| 某策略无敌 | 查极端策略、正反馈和机会成本 | 平均数掩盖策略生态 |
| 玩家不知道做什么 | 检查意图—行动—反馈闭环 | 美观不能替代可理解性 |
| 节奏一直很满 | 画分形兴趣曲线，加入呼吸和变化 | 恒定刺激会迅速疲劳 |
| 故事选择没意义 | 检查状态、知识、关系或代价是否改变 | 分支数量不是后果 |
| 公会没人留下 | 设计身份、重复相遇、互惠、规范和治理 | 在线人数不等于社群 |
| 想上新技术 | 验证技术独有价值、成熟度和回退路径 | 新奇不等于基础性价值 |
| 商业转化很好但投诉多 | 同时画价值流、资金流和影响责任链 | 短期收入可能透支信任 |

## 默认流程

1. 写一个具体玩家瞬间。
2. 写唯一最大风险假设。
3. 选 3–5 个能推翻假设的透镜。
4. 做可丢弃、可失败的最小原型。
5. 记录事实／解释／反例／下一决定。
6. 继续、修改、替代或停止。
